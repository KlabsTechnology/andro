Javascript In The Browser

Javascript is used in the browser to generally enhance the user experience. The first goal is to equal the power of desktop apps and the second is to surpass them. To this end, Javascript is used specifically to:

  • Fetch pages.

  • Send database commands via Ajax.

  • Receive a page description and render the HTML.

The Libraries and Classes

All libraries and code are stored in the file 'x4.js'.

All function prototype begin with the prefix 'x4'. Class methods do not begin with 'x4'. Free-standing functions also begin with 'x4'. All functions follow the standard javascript naming convention of beginning with a lowercase letter having subsequent words start with a capital. We consider the 'x4' prefix to be the lowercase letter when it is used, so we have 'x4ExampleFunctionName' instead of 'x4exampleFunctionName'.

The libraries are:

  • The general library, contains language extensions or shortcuts like byId()

  • Compatibility library, contains functions that smooth over the differences between IE and Firefox.

  • Framework library (except for Ajax), that does things like display messages, control popups, and any other utilities that are specific to our HTML elements.

  • Framework Ajax Library.

  • A “tabbar” class that holds “tab” objects and performs tasks related to picking tabs, making them visible, and so forth.

  • A “tabDetail” class that provides all default framework behavior for a detail tab of inputs.

  • A “tabBrowse” class that provides all default framework behavior for a search/browse. This class also controls the individual INPUT elements on a browse, there are no classes for those.

  • A (PROPOSED) “tabCustom” class that would allow for custom type classes. However, it may be possible that the detail class can be extended to do this.
