Framework URL Parameters

The framework uses a handful of common parameters for sending requests from the browser to the server.

There are three significant parameters that tell the universal dispatcher to branch away from a normal page:

  • gp_dropdown tells the universal dispatcher to return results specially formatted for the Dynamic Dropdown control.
  • gp_sql tells the dispatcher to execute a query and return a [[Rows Array]] as a serialized string. Expects to be called with a function like PHP file_get_contents . Note our section on [[SQL Injections]].
  • gp_ajaxsql tells the dispatcher to execute a particular update of a column and row based on other parameters and to return no results unless there is an error.
  • all other requests are assumed to be normal page requests and will look a value of gp_page.

The parameters that affect a normal page request are:

  • gp_page The page or table to access. (See Pages, Classes and Tables.
  • gp_xajax. A flag that tells the x_table2 class to return AJAX results instead of a complete page.
