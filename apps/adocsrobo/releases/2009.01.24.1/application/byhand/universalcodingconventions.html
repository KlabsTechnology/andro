Universal Coding Conventions

There are many coding conventions that work well in multiple languages. The conventions listed here are used in both our Javascript and PHP except where noted.

Indentations

We use a four-space indentation. We do not use the TAB character for indentations.

Curly Braces

An opening curly brace is always put on the same line as the block command that opens it.

A close brace is always alone on a line.

In keeping with the first two statements, an “else” has a trailing open curly brace:

if (some condition) {

...code goes here

}

else {

...code goes here

}

Treat Parentheses Like Curly Braces

When a function call or array definition spans multiple lines, treat the open and close parentheses like curly braces, putting the open parenthesis on the first line, and the close on a line by itself:

$myArray = array(

.....

.....lots of elements

.....

);

$myReturn = FunctionCall(

.....

.....lots of elements

.....

);

Functions Less Than 25 Lines

Be very suspicious of functions that span more than 20 lines and go to more than 4 levels of indentation. These should probably be broken up into helper functions for readability.

Put Delimiters at Front of Line

When a program command or statement will span several lines, and the portions of the code are separated by a delimiter such as a comma, put the delimiters at the beginning of each line, as in:

SELECT column1

,column2

,column3

FROM table...

or this example of PHP:

$myValues = array(

'key1'=>'value1'

,'key2'=>'value2'

,'key3'=>'value3'

);

Use Javascript Capitalization Conventions

Javascript names functions beginning with a lowercase letter, with subsequent words starting with a capital letter, and no underscores in function names. So we get 'exampleFunctionName()'. As of July 2007 we adopted that convention in both Javascript and PHP.

We also use this convention for classes and files.

HTML ID, Not Name

Library code never assigns or uses the HTML “name” attribute. The “id” attribute is used universally, and is always unique.

Use The Same Name

All of the following elements would use the same name:

  • An HTML input ID

  • A javascript variable holding the value of an HTML input.

  • A PHP variable holding a passed value, or sending one back.
