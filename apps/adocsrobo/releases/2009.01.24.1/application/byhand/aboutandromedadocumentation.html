About Andromeda Documentation

Good documentation is second only to masterful design in making a good program. The best system in the world is only as useful as its documentation, and so the Andromeda philosophy is to strive for the best possible documentation.

Our gold standard for documentation is the online PHP docs. Every nook and cranny of Andromeda should be documented with the same thoroughness that we see in the PHP docs.

The Source and The Destination

I am most comfortable using a Word Processor to generate technical documentation, and OpenOffice.org has provided a Word Processor that is Good Enough in all respects. So I use OpenOffice.org for the tutorials and explanations, and embedded docs in the source code for reference:

The final product for page-by-page viewing is HTML with CSS styling. The final product for a complete set of docs is PDF.

What We Mean By Page

The term page refers to the distinct content the user sees on the screen when they click a link. If they click on the Widgets link we say they will go to the widgets page.

How To Write Docs in OO.org

What To Put In It

Use Heading 1 to denote a distinct page. Everything before the first is discarded Heading 1, and each subsequent Heading 1 will break to a new page.

Use Heading 2 and Heading 3 within a document. No further nesting of header-level styles is defined in the style sheet.

Modify the styles in OO.org to your own liking, it will not affect the display on the site.

Make a style called code and use it for code snippets.

Make a style called parms. At some point each page needs a parms section that defines the parent page of that page.

Empty elements and page breaks are not sent to the final output, so feel free to use page breaks and blank lines as needed.

Where to Save

Application documents are stored in the 'docsapp' directory of the application.

The Last Step

When you have a document ready to be published, export it to xhtml using OO.org's built-in exporter. The Andromeda documentation generator only looks for the xhtml files, it ignores everything else.
