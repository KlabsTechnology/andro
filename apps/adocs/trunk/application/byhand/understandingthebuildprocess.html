title:Understanding The Build Process parent:Database Programming sequence:50 aliases:build

The Andromeda programmer spends a lot of time working on the database specification, which is a plaintext file in the YAML format. However, direct changes made to this file have no effect until the programmer runs a build.

In a typical iteration, the programmer makes a few changes and additions to the database spec, runs a build, and then turns to whatever coding chores are at hand.

The build process is always run through the Node Manager screen, which shows a list of all of your applications, with a link to build each one:
